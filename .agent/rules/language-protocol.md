# LANGUAGE AND DOCUMENTATION STANDARD

## 1. Strictly English Requirement
- **Code:** All variables, functions, classes, and constants must be named in English.
- **Comments:** All inline comments and JSDoc documentation must be written in English.
- **Documentation:** All markdown files, including logs and READMEs, must be written in English (except for the 'Human Specification' field in the logs, which may mirror the user's input).
- **Git:** All commit messages must be written in English.

## 2. Technical Accuracy
- Use clear, professional, and concise English.
- Avoid slang or informal abbreviations in comments.
- Ensure JSDoc tags (@param, @returns, etc.) follow standard English technical writing conventions.
